# 质量检查清单

只执行与当前文档相关的条目，但不要跳过结构、语义和可访问性检查。

## 读者与任务

- [ ] 已说明目标读者、适用范围、前置条件和预期结果。
- [ ] 页面具有一个主要目的：教程、操作指南、参考或解释。
- [ ] 开头让读者知道为什么要读以及读完能做什么。
- [ ] 事实、推断、建议和待确认项没有混写。

## 层级与语言

- [ ] 只有一个 H1，标题层级连续且能独立描述主题。
- [ ] H2 按读者执行或决策顺序排列，不按组件类型堆砌。
- [ ] 术语和缩写首次出现时定义，后文命名一致。
- [ ] 段落先给结论或动作，句子具体、主动、无空泛套话。
- [ ] 列表有引导句，各项结构平行；只有一个项目时不用列表。

## 语义结构

- [ ] 引用块只包含来源原话或必须保留的原始措辞。
- [ ] Callout 会影响读者行为或判断，而不是普通段落换背景。
- [ ] 表格用于真正的二维数据，并含语义明确的表头。
- [ ] 代码、公式、图表和图片没有互相冒充。
- [ ] 没有为“显得丰富”而增加无信息价值的卡片或图表。

## 公式

- [ ] 公式由文档源显式使用 `Formula`，没有运行时文本猜测。
- [ ] LaTeX 语法通过 KaTeX 构建验证，中文概念使用 `\text{}`。
- [ ] 运算符使用 `\times`、`\div`、`\approx`、`\le`、`\ge` 等正确命令。
- [ ] 领域公式具有读者可理解的 `aria-label`。
- [ ] 正文解释变量、单位、假设和结论；公式不是孤立装饰。
- [ ] 配置赋值、代码表达式和普通口号没有被误作公式。

## 代码与接口

- [ ] 行内代码只标记标识符、路径、字段、命令或用户输入。
- [ ] 代码块标注语言，示例最小、可复制，并说明预期结果。
- [ ] 示例不包含真实密钥、个人信息或不可公开地址。
- [ ] 接口名称、字段、默认值和版本已从当前项目确认。

## 表格、图表与媒体

- [ ] 表头与数据单元格关系清楚，表格不用于布局。
- [ ] 图表类型与关系问题匹配，正文说明读图结论。
- [ ] 节点标签完整，内部 ID、边、分支、循环和方向正确。
- [ ] 节点、Group、边标签、线段和箭头互不遮挡；编号消息按时间顺序可连续追踪。
- [ ] 超过复杂度门槛的架构图已经按边界、依赖、时序、状态或部署问题拆分。
- [ ] 图表或图片具有符合用途的可访问名称或替代文本。
- [ ] 颜色不是传达状态、风险或分类的唯一方式。
- [ ] 没有用截图替代可复制的文字、代码、表格或公式。

## 构建与视觉

- [ ] 文档构建、类型检查、链接检查和项目测试通过。
- [ ] 桌面与窄屏均检查过标题、表格滚动、代码和公式溢出。
- [ ] 浅色与深色主题下内容可读，焦点与键盘操作可用。
- [ ] 代码复制、图表查看或编辑等交互没有控制台错误。
- [ ] 未执行的验证已在交付说明中明确列出。

## 依赖边界

- [ ] 内容项目没有复制 Docs Engine CSS、解析器或组件实现。
- [ ] 内容项目没有添加基于文本特征的兼容转换。
- [ ] 共享缺口在 Docs Engine 中解决，并由包版本向依赖项目交付。
- [ ] 宿主只保留框架启动、内容发现和业务数据职责。
